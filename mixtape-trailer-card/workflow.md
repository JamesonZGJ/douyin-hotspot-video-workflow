# 官方预告情绪卡工作流

## 适用场景

适合游戏、电影、剧集的官方预告再剪。重点不是解说，而是把官方预告剪成抖音能看的情绪卡。

## 固定结构

| 段落 | 内容 |
|---|---|
| 身份开场 | 用一句话告诉观众主角是谁、处境是什么 |
| 台词翻译 | 有对白时只翻关键句；没对白时用中文故事卡补齐人物目标和情绪逻辑 |
| 故事快切 | 用 1 到 3 秒短镜头讲人物目标和冲突 |
| 背书卡 | 电影奖项可换成媒体评分、用户评价、媒体短评 |
| 片名收尾 | 只保留片名或核心情绪，不加保存视频自带结尾 |

## 执行步骤

1. 找官方素材，优先 YouTube 2K；如果被验证挡住，再用 Steam、官网、PlayStation、Xbox 官方 CDN。
2. 下载最高可用清晰度。
3. 用 `faster-whisper` 转写英文台词。
4. 挑关键台词，不逐句全翻，只翻会推动情绪和故事的句子。
5. 优先使用一条官方预告原声音轨贯穿全片，避免画面快切导致原声断裂。
6. 如果必须拆对白和音乐，先确认不会破坏歌曲完整性。
7. 音乐只能是一条连续线；不要把多段预告音乐硬拼在一起。
8. 用 FFmpeg 预切静音画面片段，避免 HyperFrames 渲染时反复跳长视频。
9. 如果声音来自同一条连续预告，画面也按成片时间从同一条源视频顺序切；相邻片段的源时间不能重叠。
10. HyperFrames 画布固定 `1080x1920`，游戏画面用中间 `9:10` 框，左下角放中文名，右上角放工作室。
11. 跑 `lint / validate / inspect`，确认字幕不溢出。
12. 渲染 MP4，检查时长、分辨率、音轨和关键帧。

## 字幕原则

- 中文字幕要像预告文案，不要像逐字机翻。
- 有英文对白时，中文字幕优先翻意思。
- 没有对白时，用中文故事卡补齐人物是谁、要去哪、为什么出发、真正探索什么。
- 台词字幕按说到的节奏逐字出现。
- 不做中文旁白。

## 声音原则

- 不直接使用每条画面片段的原声。
- 白话：视频负责画面，音轨单独做。
- 背景音乐必须连续，不能一刀一断。
- 优先使用同一条官方预告原声；对白只保留听得清、能推动故事的原音。
- 结尾自然收音，不再加额外口播。

## 评分卡原则

- `IGN 10/10`、`Steam 好评如潮` 这类文案必须有当天来源。
- Steam 是 `Very Positive` 时写“特别好评”，不能写“好评如潮”。
- 没有评分时，用媒体短评，不编分数。

## 画面原则

- 画面框可以试 `9:10`，比横版更适合手机，也比铺满屏更容易看清主体。
- 相邻快切片段不能从源视频重叠取材；宁可顺着原预告往后走，也不要倒回同一个镜头。
- 不加“官方预告二剪”这类说明文字。
- 不加保存视频留下的结尾提示。
- 首屏不要加灰色蒙层，先让观众看清真实画面。
