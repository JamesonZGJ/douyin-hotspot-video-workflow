# 提问表

按顺序问。用户已经明确说过的内容不要重复问。

目标是先把视频需求定义准确，再生成工程脚手架或剪辑计划。

## 必问 12 项

| 序号 | 问题 | 示例答案 |
| --- | --- | --- |
| 1 | 你想做的视频主题是什么？ | 任天堂直面会总结 / 某款游戏预告 / 游戏体验复盘 |
| 2 | 准备发布到什么平台？ | 抖音 / 小红书 / B站 / 视频号 / TikTok / YouTube Shorts |
| 3 | 希望视频比例是什么？ | 9:16 竖屏 / 16:9 横屏 / 1:1 方屏 / 4:5 |
| 4 | 希望视频大概多长？ | 15 秒 / 30 秒 / 60 秒 / 90 秒 / 2-3 分钟 |
| 5 | 声音怎么处理？ | AI 旁白 / 自己录音 / 原视频原声 / 无旁白只做字幕卡 / 原声 + 中文字幕 |
| 6 | 视频表达更偏哪种？ | 快报型 / 情绪型 / 观点型 / 种草型 / 吐槽型 / 预告片型 / 极简卡片型 |
| 7 | 这条视频的主要目标是什么？ | 快速讲清楚 / 引发讨论 / 安利作品 / 做观点表达 / 提高完播率 |
| 8 | 主要给谁看？ | 路人 / 核心玩家 / 粉丝 / 行业从业者 / 没了解过这个话题的人 |
| 9 | 有没有必须出现的 1-3 个重点？ | 游戏名、关键系统、发售日、争议点、最强画面 |
| 10 | 有没有不能出现的内容或禁忌？ | 剧透 / 未证实爆料 / 负面词 / 版权风险素材 / 低俗梗 |
| 11 | 字幕和画面文字希望怎样？ | 全程字幕 / 重点大字 / 中英双语 / 少字高级感 / 信息卡片密集型 |
| 12 | 是否需要同时生成封面标题、发布标题、简介文案和话题标签？ | 需要 / 不需要 / 只要标题和封面 |

## 默认值

只有用户没给、且任务风险不高时才用默认值。

| 项目 | 默认值 |
| --- | --- |
| 平台 | 抖音 |
| 比例 | 9:16 竖屏 |
| 时长 | 60-90 秒 |
| 声音 | AI 旁白 |
| 表达 | 快报型 |
| 目标 | 快速讲清楚 |
| 受众 | 没了解过这个话题的人 |
| 字幕 | 全程字幕 + 重点大字 |
| 发布物料 | 生成封面标题、发布标题、简介和话题标签 |

## 平台和比例

| 平台 | 常用比例 | 备注 |
| --- | --- | --- |
| 抖音 | 9:16 | 竖屏优先，节奏要快 |
| 小红书 | 4:5 / 9:16 | 文字信息和封面更重要 |
| B站 | 16:9 / 9:16 | 横屏适合长视频，竖屏适合动态流 |
| 视频号 | 9:16 | 开头要更直给 |
| TikTok | 9:16 | 少字、强画面、强节奏 |
| YouTube Shorts | 9:16 | 标题和前 3 秒很重要 |

## 声音处理

| 声音方案 | 处理方式 |
| --- | --- |
| AI 旁白 | 先定语气，再生成口播 |
| 自己录音 | 录音时长决定最终时长，不用 TTS 顶替 |
| 原视频原声 | 保留原声节奏，字幕负责讲清楚 |
| 无旁白只做字幕卡 | 文字卡承担叙事，镜头要更连贯 |
| 原声 + 中文字幕 | 不切碎原声，字幕贴着原声情绪走 |

## 表达类型

| 类型 | 处理方式 |
| --- | --- |
| 快报型 | 爆点先行，信息密度高，结尾一句判断 |
| 情绪型 | 少讲事实，多保留音乐、停顿和镜头呼吸 |
| 观点型 | 开头先给结论，再用事实支撑 |
| 种草型 | 重点讲“为什么值得看/玩/买” |
| 吐槽型 | 判断要明确，但事实不能乱说 |
| 预告片型 | 原声和画面优先，文字只补关键信息 |
| 极简卡片型 | 少字高级感，信息只留最关键的几句 |

## 停下来确认的情况

- 官方中文名不确定。
- 发售日期、平台、DLC、免费更新不确定。
- 用户素材里包含多个版本，不知道该用哪条。
- 用户要求模仿具体博主，但没有说明是学标题、口播还是剪辑节奏。
- 用户要求上传、预测或复盘；这些不属于本 skill，需另行确认是否调用其他流程。
